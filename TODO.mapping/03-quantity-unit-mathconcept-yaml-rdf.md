# 03 — Domain classes with bidirectional YAML + RDF mappings

## Goal

Rewrite `Isq::Quantity`, `Isq::Unit`, and `Isq::MathConcept` with full
`yaml do` and `rdf do` / `turtle do` mappings so they support:

- `Quantity.from_yaml(yaml_string)` → array of Quantity instances
- `quantity.to_turtle` → Turtle string
- `Quantity.from_turtle(ttl_string)` → Quantity instance
- `quantity.to_yaml` → YAML string

This replaces the procedural export logic with fully model-driven serialization.

## Rationale

The current domain classes have `rdf do` blocks that only cover a subset of the
predicates generated by the procedural rake task. They have no `yaml do` blocks
at all, so they can't deserialize the source YAML.

The YAML source structure (from `iso-iec-80000/sources/dataset/`):

```yaml
- part: '3'
  edition: '2019'
  id: t3-1.1
  num: 3-1.1
  designations:
  - designation:
      en:
        text: length
        index_as: [length]
  symbols: [l, L]
  def:
    en: linear extent in space between any two points
  units:
  - en: metre
    symbol: [m]
  remarks:
    en: |
      Length does not need to be measured along a straight line.
```

The target Turtle:
```turtle
isoiec80000:t3-1.1 a isoiec80000:Quantity, smart:TermEntry ;
  dcterms:identifier "3-1.1" ;
  skosxl:prefLabel isoiec80000:term-t3-1.1-0 ;
  skosxl:altLabel isoiec80000:sym-t3-1.1-0 ;
  skosxl:altLabel isoiec80000:sym-t3-1.1-1 ;
  skos:definition """linear extent..."""@en ;
  skos:note """Length does not..."""@en ;
  isoiec80000:hasUnit isoiec80000:unit-m ;
  smart:hasBindingnessType smart:normative ;
  dcterms:isPartOf isoiec80000:part-3 .
```

## Design decisions

### Custom YAML methods for structural mismatch

The YAML structure has deeply nested multilingual fields (`def.en`, `remarks.en`,
`designations[].designation.en.text`) that don't map cleanly to flat attributes.
Use `with: { from:, to: }` in the `yaml do` block for these fields.

Keep the custom methods minimal — they only adapt structure, not semantics.

### Multilingual text as `Lutaml::Rdf::Literal`

Attributes like `definition` and `note` carry both text and language.
Use `Lutaml::Rdf::Literal` so the `lang_tagged: true` predicate in the RDF
mapping can extract the language tag bidirectionally.

### `members` for Term instances

The `rdf do` block uses `members :designations` and `members :symbols` to
generate separate graph nodes (the `smart:Term` instances) in the Turtle
output. This is the lutaml-model mechanism for composite RDF structures.

### skosxl:prefLabel / skosxl:altLabel generation

The first designation is `skosxl:prefLabel`, the rest plus all symbols are
`skosxl:altLabel`. This is a positional rule — handle via a computed method
or a custom predicate mapping that inspects the index.

### Part and edition metadata

`part` and `edition` are YAML-only fields not directly serialized in the entry's
Turtle. They are used by the Dataset to group entries and generate
PublicationDocument instances. Keep them as attributes with YAML-only mappings.

## Implementation

### File: `lib/isq/quantity.rb` (rewrite)

```ruby
module Isq
  class Quantity < SduSmart::TermEntry
    attribute :identifier, :string
    attribute :pref_label, :string
    attribute :notation, :string, collection: true
    attribute :definition, :string
    attribute :note, :string
    attribute :has_unit, :string, collection: true
    attribute :designations, Isq::Designation, collection: true
    attribute :symbols, Isq::SymbolTerm, collection: true
    attribute :part, :string
    attribute :edition, :string

    yaml do
      map :id, to: :id
      map :num, to: :identifier
      map :part, to: :part
      map :edition, to: :edition
      map "def", with: { from: :definition_from_yaml, to: :definition_to_yaml }
      map :remarks, with: { from: :note_from_yaml, to: :note_to_yaml }
      map :designations,
          with: { from: :designations_from_yaml, to: :designations_to_yaml }
      map :symbols,
          with: { from: :symbols_from_yaml, to: :symbols_to_yaml }
      map :units, with: { from: :units_from_yaml, to: :units_to_yaml }
    end

    rdf do
      namespace Isq::Rdf::Namespaces::IsoIec80000Namespace,
                SduSmart::Rdf::Namespaces::SmartNamespace,
                Lutaml::Rdf::Namespaces::DctermsNamespace,
                Lutaml::Rdf::Namespaces::SkosNamespace,
                Isq::Rdf::Namespaces::SkosxlNamespace

      subject { |m| "isoiec80000:#{m.id}" }
      type "isoiec80000:Quantity"

      predicate :identifier, namespace: DctermsNamespace, to: :identifier
      predicate :prefLabel, namespace: SkosNamespace, to: :pref_label
      predicate :notation, namespace: SkosNamespace, to: :notation
      predicate :definition, namespace: SkosNamespace, to: :definition,
                lang_tagged: true
      predicate :note, namespace: SkosNamespace, to: :note, lang_tagged: true
      predicate :hasUnit, namespace: IsoIec80000Namespace, to: :has_unit
      predicate :hasBindingnessType, namespace: SmartNamespace,
                to: :bindingness_type
      predicate :isPartOf, namespace: DctermsNamespace, to: :is_part_of

      members :designations
      members :symbols
    end

    # --- YAML structural adapters ---

    def definition_from_yaml(hash)
      return unless hash
      @definition_lang = hash.keys.first.to_s
      hash.values.first
    end

    def definition_to_yaml(value)
      return unless value
      { (@definition_lang || "en") => value }
    end

    def note_from_yaml(hash)
      return unless hash
      @note_lang = hash.keys.first.to_s
      hash.values.first
    end

    def note_to_yaml(value)
      return unless value
      { (@note_lang || "en") => value }
    end

    def designations_from_yaml(raw)
      return [] unless raw
      raw.each_with_index.map do |d, i|
        lang_data = d["designation"] || d[:designation]
        lang = lang_data.keys.first
        text_data = lang_data[lang]
        Isq::Designation.new(
          id: "term-#{id}-#{i}",
          text: text_data["text"],
          lang: lang.to_s,
          index_as: text_data["index_as"],
          term_form_type: "smart:fullForm",
        )
      end
    end

    def designations_to_yaml(designations)
      designations.map do |d|
        { "designation" => { d.lang => { "text" => d.text, "index_as" => d.index_as } } }
      end
    end

    def symbols_from_yaml(raw)
      return [] unless raw
      raw.each_with_index.map do |sym, i|
        Isq::SymbolTerm.new(
          id: "sym-#{id}-#{i}",
          text: sym,
          lang: "en",
          term_form_type: "smart:symbol",
        )
      end
    end

    def symbols_to_yaml(symbols)
      symbols.map(&:text)
    end

    def units_from_yaml(raw)
      return [] unless raw
      raw.map do |u|
        sym = u["symbol"]&.first
        sym ? "isoiec80000:unit-#{sym}" : "isoiec80000:unit-#{u['en']&.downcase&.gsub(/\s+/, '-')}"
      end
    end

    def units_to_yaml(unit_refs)
      # Lossy: unit refs back to {en:, symbol:} requires Dataset lookup
      # For round-trip, Dataset must provide unit name resolution
      []
    end
  end
end
```

### File: `lib/isq/unit.rb` (rewrite)

```ruby
module Isq
  class Unit < SduSmart::TermEntry
    attribute :pref_label, :string
    attribute :notation, :string, collection: true

    yaml do
      map :en, to: :pref_label
      map :symbol, to: :notation
    end

    rdf do
      namespace IsoIec80000Namespace, SmartNamespace, DctermsNamespace, SkosNamespace
      subject { |m| "isoiec80000:#{m.id}" }
      type "isoiec80000:Unit"
      predicate :prefLabel, namespace: SkosNamespace, to: :pref_label
      predicate :notation, namespace: SkosNamespace, to: :notation
      predicate :hasBindingnessType, namespace: SmartNamespace, to: :bindingness_type
    end
  end
end
```
